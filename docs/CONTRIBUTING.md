# Contributing — DentalFlow / dentalapp

## Reglas
1) Todo cambio va por PR (branch por PR).
2) No se empuja directo a `main` salvo hotfix aprobado.
3) Commits pequeños y descriptivos (convención sugerida):
- `chore: ...`
- `feat: ...`
- `fix: ...`
- `docs: ...`
- `db: ...`
4) Antes de abrir PR:
- `npm run build`
- `npm run lint` (si existe)
5) Todo PR actualiza `docs/PR_LOG.md`.
6) Decisiones nuevas o cambios de reglas → `docs/DECISIONS.md`.
